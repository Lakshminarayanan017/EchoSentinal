# echoSentinel v2

Underwater sound event detection and classification for **PS-12: AI tools for Underwater
Domain Awareness**. Detects sound events in hydrophone `.wav` recordings — robust to heavy
engine background noise and very faint (low-SNR) events — and classifies them into the four
PS-12 categories, emitting the competition JSON format.

| ID | Class | Meaning |
|----|-------|---------|
| 1 | `vessel` | Ship engine drone, propeller cavitation |
| 2 | `marine_animal` | Whale calls, dolphin whistles/clicks, … |
| 3 | `natural_sound` | Rain, earthquakes, waves, ice, volcanoes |
| 4 | `other_anthropogenic` | Sonar pings, airguns, pile driving, chains, explosions |

## Design in one paragraph

All audio is standardized to 32 kHz mono float32 by a format-agnostic loader
(8→64-bit PCM/float, any sample rate, huge files read in streamed blocks). A frame-level
sound-event-detection model (PCEN front-end + pretrained PANNs CNN14 backbone, CRNN as the
light alternative) outputs per-class posteriors every 10 ms. Training data is generated by an
online **scene synthesizer** that mixes clean labeled events into engine-noise/ambient beds at
controlled low SNRs — closing the domain gap that killed the v1 prototype. Post-processing
(median filter → per-class hysteresis thresholds → gap merging → non-overlap enforcement →
second-rounding) turns posteriors into competition-valid events, tuned against the official
metric (Identification Error Rate, where a missed event costs 4× a false alarm). Everything is
validated locally on a synthetic set scored with pyannote-metrics IER.

## Repo layout

- `src/echosentinel/` — the installable package (audio IO, features, data/synthesizer, models, training, inference, evaluation)
- `scripts/` — numbered pipeline steps (audit → manifests → noise bank → synth val set → train → evaluate → predict)
- `configs/` — OmegaConf YAMLs; no magic numbers in code
- `predict.py` — single competition entrypoint (`--input_dir --output_json`), docker target
- `tests/` — pytest suite (format coverage, non-overlap invariant, JSON schema, streaming equivalence)
- `weights/` — model artifacts (git-ignored)
- The old v1 prototype lives untouched in `../echosentinal/` for reference only.
